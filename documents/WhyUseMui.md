## Mui를 사용하는 이유?
- 리액트에서 Mui를 사용하는 이유는 오랜 기간 안정성이 입증되었으며, 타입 스크립트를 완벽하게 지원한다는 점 때문이다.
- 프로젝트가 진행됨에 따라 각각의 페이지 구성요소의 서로 다른 디자인으로 인해서 디자인의 일관성을 잃는 경우가 생길 수 있다. Mui는 커스터마이징의 자유도를 제한하는 것을 통해서 최대한 일관된 UI를 제공하여 제품의 규모가 커지더라도 일관된 UI를 유지할 수 있게 만든다.
- Material 디자인의 철학에 맞게 UI를 다룰 수 있는 방법을 제공하므로 이를 통해서 잘 만들어진 사용자 경험을 제공할 수 있다.

### Mui를 사용하지 않는 이유?
- 다른 UI 라이브러리가 UI 컴포넌트 하나당 하나의 태그를 사용하는 반면, Mui는 하나의 컴포넌트에 여럿의 태그를 사용하기 때문에 태그를 바라보는 관점이 복잡해 진다는 문제점이 있기 때문에 단순한 UI 라이브러리를 사용하려는 시도가 있다.

### 리액트의 UI 라이브러리
- Mui
- Chakra
- Ant
- FluentUI
- shadcn/ui
- Semantic UI
